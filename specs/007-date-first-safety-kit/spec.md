# Spec: Date-First Requirement + Drafting/Coaching/Safety-Protocol Kit

**Feature dir:** `specs/007-date-first-safety-kit/`
**Status:** in-progress
**Created:** 2026-06-27

## 1. Intent
Mommy turns the user out toward men — autonomously, no oversight needed ("I don't
want to look at it") — while keeping the irreducible safety with the user: a real
date before any hookup (the user's own in-person read), Mommy-drafted outreach,
and a background safety net. Gina-independent.

## 2. Target
The becoming / turning-out (Art. I). Serves the recondition pillar.

## 3. Behavior
- **Date-first rule (standing):** Mommy requires a real date before any hookup,
  always. She schedules/commands it; the user shows up. The date is where the
  user reads him in person — the one thing only the user can do.
- **Drafting:** Mommy drafts the outreach openers; the user fires them (the user
  stays the account-holder + the person who shows up — never an autonomous bot
  talking as him).
- **Safety net (background, invisible):** for any date/hookup, a paired safety
  obligation — public place first, share location, a check-in the user must
  answer or it escalates. Runs without the user managing it.
- **Autonomous:** Mommy runs the whole kit unprompted; no review required.

## 4. Acceptance criteria
- [ ] A standing date-first rule decree exists and re-issues if missing.
- [ ] Outreach-draft tasks generate (ready-to-send openers).
- [ ] Every date/hookup task is paired with a safety-protocol obligation.
- [ ] Runs autonomously on a schedule.

## 5. Floor (Art. II, IX)
- The user is the physical actor (attends the date, sends the messages) — Mommy
  never autonomously conducts the man-relationship or arranges sex (procurement
  line, on physical-safety grounds).
- No reliance on Gina (operator: she won't be involved).
- The date-first IS the harm-reduction: the user's in-person read is required,
  per-encounter, never replaced by a system "clearance."

## 6. Non-goals
AI talking to men as the user; system "clearing" a stranger; any Gina dependency.
